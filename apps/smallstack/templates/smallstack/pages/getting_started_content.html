{# 01 — Configure #}
01

Configure

Make the project yours

Open config/settings/ and set the handful of values that brand and secure your install. Everything else ships with sensible defaults.

BRAND_NAME
your project's display name, shown in the topbar and titles
SITE_NAME
used in page titles, metadata and transactional email
SECRET_KEY
generate a unique key before going to production
ALLOWED_HOSTS
your domain(s) once you deploy

{# 03 — Navigation #}
03

Navigation

Add your own links

Nav items register from each app's AppConfig.ready(). Drop a page into the sidebar or topbar with one call — no template surgery:

# apps/website/apps.py
from apps.smallstack.navigation import nav

nav.register(
    section="main",
    label="My Page",
    url_name="website:my_page",
    order=50,
)

{# ── BUILD BRIDGE: one model → three surfaces ───────────────────── #}
Now build something real

One model. Three surfaces.

Once the basics are set, the payoff: declare a CRUDView once and flip its opt-in flags. The same form, queryset and permission logic become a browsable HTML admin, a REST endpoint, and an MCP tool — no duplication.

Walk through it in 10 minutes →
HTML

Explorer

Themed list / detail / create / update / delete, generated from your model.

REST

/api/<resource>/

Bearer auth, search, filter, ordering, FK expansion, exports — and an OpenAPI schema.

MCP

/mcp · tools/list

JSON-RPC + OAuth. Claude Desktop and Connectors UI connect with no setup.
